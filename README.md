### Тест-кейсы:

#### TC-001: Успешная авторизация

| Поле | Значение |
|------|----------|
| **Название** | Авторизация с корректными учетными данными |
| **Предусловие** | Пользователь не авторизован в системе |
| **Шаги** | 1. Открыть страницу авторизации GitHub<br>2. Ввести корректный логин<br>3. Ввести корректный пароль<br>4. Нажать кнопку "Sign in" |
| **Ожидаемый результат** | Пользователь переходит на главную страницу |
| **Фактический результат** | ✅ Вход выполнен успешно |
| **Статус** | PASSED |

#### TC-002: Неуспешная авторизация

| Поле | Значение |
|------|----------|
| **Название** | Авторизация с некорректными учетными данными |
| **Предусловие** | Пользователь не авторизован в системе |
| **Шаги** | 1. Открыть страницу авторизации GitHub<br>2. Ввести некорректный логин<br>3. Ввести некорректный пароль<br>4. Нажать кнопку "Sign in" |
| **Ожидаемый результат** | Пользователь остается на странице авторизации, отображается сообщение об ошибке |
| **Фактический результат** | ✅ Сообщение об ошибке "Incorrect username or password" появилось |
| **Статус** | PASSED |

#### TC-003: Переход к репозиторию

| Поле | Значение |
|------|----------|
| **Название** | Переход к репозиторию из профиля пользователя |
| **Предусловие** | Пользователь авторизован в системе, открыта страница профиля |
| **Шаги** | 1. Открыть страницу профиля<br>2. Найти репозиторий "china"<br>3. Кликнуть по ссылке репозитория |
| **Ожидаемый результат** | Выполнен переход на страницу репозитория |
| **Фактический результат** | ✅ Переход выполнен на https://github.com/varvara12348/china |
| **Статус** | PASSED |
